# retux kernel architecture

this document describes the high‑level design, boot sequence, and core subsystems of the retux kernel.

---


## 1. boot Process (32‑bit -> 64‑bit)

1. **GRUB2** loads the kernel via Multiboot2 and jumps to the 32‑bit entry point `start` (defined in `boot.asm`).
2. the boot stub performs sanity checks:
   - multiboot magic number (`0x36D76289`)
   - CPUID support
   - long‑mode support (via `CPUID` leaf `0x80000001`, bit 29 of EDX)
3. **page tables** are set up identity‑mapping the first 4 GiB of physical memory using 2 MiB huge pages (PML4 -> PDPT -> PD with huge‑page bit set).
4. **paging is enabled**:
   - CR3 points to the PML4
   - PAE enabled (CR4.PAE)
   - LME and NXE enabled in EFER
   - PG set in CR0
5. a minimal **GDT** is loaded (null, code, data, and a placeholder for the TSS descriptor).
6. a far jump loads the 64‑bit code segment and execution continues in `long_mode_start`.
7. data segments are reloaded and `kernel_main` (C++) is called with the Multiboot info address in RDI.

---


## 2. memory Management

### physical memory manager (`pmm`)
- tracks all physical memory up to 4 GiB using a **bitmap** (128 KiB).
- on init, reads the Multiboot2 memory map (tag type 6) and marks usable regions as free.
- reserves:
  - the kernel image (from `_kernel_start` to `_kernel_end`)
  - the first 1 MiB (legacy BIOS areas)
- provides:
  - `alloc_frame()` - returns a 4 KiB aligned physical address
  - `free_frame()` - self descriptive
  - `print_stats()` - for debugging

### virtual memory manager (`vmm`)
- manages the x86‑64 4‑level paging structure (PML4 -> PDPT -> PD -> PT).
- supports **4 KiB pages** and automatically **splits 2 MiB huge pages** when a finer mapping is needed.
- provides:
  - `map(virt, phys, flags)` - maps a single 4 KiB page; creates intermediate tables on demand.
  - `unmap(virt)` - removes a mapping (does not free physical frames).
  - `virt_to_phys(virt)` - walks the live tables and returns the physical address, handling huge pages.
  - `remap_kernel()` - replaces the boot‑time identity mapping with per‑section permissions:
    - `.text` -> `KERNEL_RX` (read + execute, no write)
    - `.rodata` -> `KERNEL_RO` (read, no write, no execute)
    - `.data` / `.bss` -> `KERNEL_RW` (read + write, no execute)

---


## 3. heap allocator (`heap`)
- implements a **simple linked‑list allocator** with coalescing (boundary tags).
- virtual address range: `0xFFFF800000000000` - `0xFFFF8000FFFFFFFF` (4 GiB total).
- each block has a header and footer (both store size + free flag). free blocks additionally store a `next` pointer in the data area.
- on `kmalloc`:
  - searches the free list for a block large enough.
  - if none found, **grows the heap** by allocating physical frames via `pmm` and mapping them via `vmm`, then creates a new free block and coalesces.
- minimum block size is aligned to 8 bytes.
- `kfree` marks the block free and coalesces with adjacent free blocks.

---


## 4. interrupt handling

### IDT (`idt`)
- sets up 256 entries; first 32 are exception handlers (generated by `isr.asm`).
- gate type: interrupt gate (present, DPL=0, 64‑bit).
- the **double‑fault** handler (#DF, vector 8) is configured with **IST1** (interrupt stack table) to avoid triple‑faults on stack corruption.
- a common C++ handler (`isr_common_handler`) prints a register dump and halts.

### TSS (`tss`)
- a task state segment (TSS) is used to provide the IST stacks.
- a dedicated 8 KiB stack is allocated for double faults (`df_stack`).
- the TSS descriptor is written into the GDT slot (defined in `boot.asm`) at runtime.
- `ltr` loads the TSS selector (`0x18`).

---


## 5. output
- **serial** (COM1, 38400 baud) - used for debug output and a `print` interface.
- **VGA** text mode (0xB8000) - 80×25, white on black, with `print` and `clear`.

---

## 6. build & link
- the **linker script** (`linker.ld`) places the kernel at physical address 1 MiB (the standard for Multiboot).
- symbols `_kernel_start/end`, `_text_start/end`, etc. are exported for the PMM and VMM.
- the boot stub is written in NASM, the rest in C++ (compiled with an `x86_64-elf` cross‑compiler).
